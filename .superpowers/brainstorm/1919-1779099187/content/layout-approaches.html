A — Filtered List

Single scrollable list with filter tabs (All / Arrived / Waiting / Called). Each card shows context-appropriate actions based on its status. Default view shows everything; tabs narrow it down.

Pros

  • Simple, compact
  • Easy to scan all patients at once
  • Familiar pattern

Cons

  • Mixed statuses can feel noisy

B — Kanban Columns

Three side-by-side columns, one per status. Cards flow left-to-right as patients progress. The workflow is visually obvious at a glance — you can see the queue depth per stage immediately.

Pros

  • Workflow is instantly readable
  • Queue depth per stage visible

Cons

  • Tight on smaller screens
  • Columns can be uneven/sparse

C — Grouped + Filtered List (Recommended)

Single list, items grouped under section headers per status, with filter tabs to isolate a group. Cards are color-tinted by status. Filters + grouping without the constraint of kanban columns.

Pros

  • Works on any screen width
  • Groups give structure without forcing columns
  • Filters still work for isolation

Cons

  • Slightly more vertical scrolling
